# Course Page Overrides

Applies to:

- `course.html`
- `index.html`
- `course-system.html`
- `case-library.html`
- `mindmap-hub.html`
- `codex.html`
- case/demo pages generated by `course_layout()`

## Home Hero

- Keep the first viewport focused on course entry and learning tracks.
- Use the orange/yellow/black pixel style for calls to action.
- The cover image can stay framed in a hard-shadow panel.
- Track cards should feel like physical blocks, not soft feature cards.

## Course Map

- Chapter cards use white surfaces with black borders.
- Chapter labels use yellow stickers.
- Lesson links should show `number-icons/NN.png` wherever possible.
- Main lesson and worksheet actions should remain visible from the card.

## Case Library

- Filters are sticker-like controls.
- Active filters use orange.
- Case status and tags use yellow sticker treatment.
- Keep result rows dense enough for scanning; do not turn the library into a marketing gallery.

## Course System Page

- This page is the operational documentation entrypoint.
- Always expose both:
  - `COURSE_TEXT_SYSTEM.md`
  - `design-system/ai-beginner-course/MASTER.md`

## Resource Hub

- Mindmap thumbnails should use hard black frames.
- Do not use blurred image cards or soft gallery shadows.
