Codex 专题 · 从会聊天到会交付

Codex 能做什么

Codex 可以理解为帮你做软件工作的 AI 助手。它不只是回答问题，还能在代码仓库里读文件、改代码、运行命令、检查结果，并把工作拆给多个会话并行推进。

先看它是什么 看能做什么
一句话理解
  1. ChatGPT 更像“问答和思考助手”。
  2. Codex 更像“能进入项目现场的执行助手”。
  3. 你给目标、上下文和验收标准，它负责读、改、跑、验收。

用第一性原则看 Codex

先拆底层工作流，再看产品名。

1. 任务你要完成什么：修 Bug、做网页、写教程、跑测试、整理报告。
2. 上下文Codex 需要看到项目文件、运行环境、已有规范和你的具体要求。
3. 工具它可以调用命令行、浏览器、文件编辑、多会话、插件和技能。
4. 产物最终不是一句回答，而是改好的文件、可打开的网站、检查结果或可复用流程。
5. 验收必须用运行结果、截图、链接检查、测试或人工标准证明完成。

Codex 常见能力

新手先记住这些实际用途。

代码与项目

读懂项目并改代码

让 Codex 先看项目结构，再按你的目标修改文件。例如修复样式、补功能、改脚本、整理目录、生成静态页面。

运行与验证

跑命令、查错误、做验收

它可以运行生成脚本、测试命令、本地服务和浏览器检查，把“改了”推进到“确实能用”。

网页与内容

把资料做成可阅读产品

适合把 Markdown、脑图、课程大纲、报告，整理成手机可读的 HTML 教程、资源页或发布目录。

多会话协作

并行探索、执行、验收

复杂任务可以让不同会话分别做内容审查、UI 验收、实现建议，主会话负责收口和最终判断。

自动化

持续检查和定时跟进

适合定期 review、提醒、监控变化、持续检查项目状态，但要明确频率、目标和输出形式。

知识沉淀

用 Skill 固化做事方法

如果某类任务会反复做，可以把步骤、检查表、参考资料沉淀成 Skill，让以后执行更稳定。

新手怎么提需求

给 Codex 的好任务长什么样

  • 说清目标：我要最终得到什么。
  • 说清范围：改哪个目录，哪些文件别碰。
  • 说清标准：怎么证明完成，比如截图、测试、链接检查。
  • 允许它先读代码，再决定怎么改。
边界意识

Codex 不应该替你决定什么

  • 不要让它无授权删除、重置或覆盖重要文件。
  • 涉及账号、付款、隐私、线上发布，要先明确授权。
  • 结果必须验收，不能只因为回答很自信就直接信。
  • 产品方向、商业取舍、最终发布责任仍然由人决定。

3 个上手练习

从小任务开始，建立“目标 -> 执行 -> 验收”的习惯。

练习 1：让 Codex 读一个项目

要求它输出入口文件、生成方式、主要页面、可能的风险，不改文件。

练习 2：让 Codex 改一个页面

给出目标用户和验收标准，让它改完后运行本地预览并截图检查。

练习 3：让 Codex 沉淀一个流程

把一次成功任务总结成检查清单，之后同类任务按清单执行。

练习 4：让多会话做验收

一个会话改内容，一个会话看移动端，一个会话查断链，最后主会话汇总。

Codex 教学 · 目标是让新手理解“它能交付什么”，而不是只会背产品介绍。
