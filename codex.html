把 Codex 当成能交付的软件工作助手来学

这一段恢复为原来的进阶课程：Codex 能做什么、Loop Engineering、网页制作、Git/GitHub 协作，以及 Dify + Codex 自动化。

老鸟进阶学习顺序
  1. 先理解 Codex 如何读、改、跑、验收。
  2. 再用 Loop Engineering 管理多轮迭代。
  3. 接着练网页制作、GitHub 发布和线上验证。
  4. 最后把 Dify 与 Codex 组合成自动化工作流。

老鸟进阶 5 节课

从 Codex 单次交付，走向循环迭代、网页发布和自动化流程。

第 1 课

Codex能做什么

理解 Codex 不是普通聊天机器人，而是能进入项目现场读、改、跑、验收的 AI 工作助手。

本课产物写一个给 Codex 的任务说明，包含目标、范围、验收标准和禁止事项。

第 2 课

Loop Engineering是什么

理解 Loop Engineering 是一种循环式工作方法：计划、执行、验收、修正，再进入下一轮。

本课产物为“把一个旧网页改成手机友好教程站”设计 3 轮 Loop，每轮写清目标和验收证据。

第 3 课

如何用Codex制作好看的网页

学会把目标用户、页面结构、视觉规范和移动端验收交给 Codex，让网页既能看又能用。

本课产物写一段网页制作需求，让 Codex 为“AI 新手课程”设计一个手机优先首页，并列出 5 条验收标准。

第 4 课

Git、GitHub与Codex协作

理解 Git 记录本地修改，GitHub 托管和发布代码，Codex 可以帮助提交、推送和验证线上结果。

本课产物写出一次 GitHub Pages 发布流程：本地生成、检查、提交、推送、线上验证，每一步写出成功证据。

第 5 课

你真的会用Dify吗？

学会用 Codex 搭建 Dify 平台、完成 Docker 部署，并把 Wiki 发布、Bug 汇总和自动 Code Review 做成自动化流程。

本课产物设计一个 Dify + Codex 自动化方案：先写 Docker 部署步骤，再画出 Wiki 发布、Bug 飞书文档、自动 Code Review 三条流程。
进入主讲义打开工作纸
好任务

给 Codex 的需求怎么写

  • 说明目标、路径、可改范围和禁止事项。
  • 要求先读上下文，再给计划，再执行。
  • 每轮改完必须运行、检查、截图或验证链接。
  • 提交发布前检查 diff、断链和线上结果。
风险边界

不要把 Codex 当无限权限自动驾驶

  • 不要无确认删除、重置、提交或发布。
  • 不要把密钥、账号和生产权限交给自动流程。
  • 不要只看回答，要看文件、命令和线上证据。
  • 不要手改生成目录而忽略源生成器。
