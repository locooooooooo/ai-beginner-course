用 Codex 写小说，并制作成视频

以《重生之我在同福客栈》第 01 集为例，把一个原创穿越故事推进到小说正文、视频脚本、分镜、结构化数据、字幕、角色设定和 MP4 review 成片。

一句话案例

Codex 不只是写文案，而是把内容推进到可交付

这个项目从一个故事想法开始，最后形成一套可以继续生产第 02 集的内容工程。核心价值不是某一段文字写得好，而是把“小说”和“视频”中间的所有转换步骤都显式化、文件化、可复用。

项目定位

做的是一集古装轻喜剧短剧样片

题材古装轻喜剧、穿越、客栈群像、微悬疑。
剧情核心现代短视频编导林舟穿进同福客栈，靠“客栈生存簿”救场并留下。
结尾钩子半页账纸上出现“葵花派”，为第 02 集埋下悬疑线。
工作流

Codex 实际做了这 8 步

  1. 1确定题材和钩子
    先明确穿越、客栈群像、轻喜剧和微悬疑，不让内容跑散。
  2. 2生成小说正文
    先写完整故事，保证人物关系、冲突和结尾统一。
  3. 3压缩视频脚本
    把正文改成适合旁白、口播和短视频节奏的脚本。
  4. 4拆镜头分镜
    按画面、动作、对白和情绪拆成可生成的视频镜头。
  5. 5整理结构化 JSON
    让每个场景都有时长、画面提示词、动作、旁白和字幕。
  6. 6补字幕和角色 bible
    用 SRT 和角色一致性设定控制节奏与人物外观。
  7. 7组合 HTML 成片工程
    用本地 HyperFrames 工程承载 15 个场景，减少外部依赖。
  8. 8导出 review 成片
    先生成 review MP4，再根据效果决定是否出高质量 final。
可迁移经验

这个案例最值得学的 5 件事

  • 先写完整小说，再拆视频，故事内核更稳。
  • 角色一致性设定要尽早写，不然画面生成容易前后变脸。
  • 结构化 JSON 比纯文本更适合进入视频流水线。
  • 先出 review，再出 final，可以降低返工成本。
  • 本地依赖优先，预览、校验和渲染更可控。
对外表达

分享时可以这样讲

我用 Codex 完成了一个“小说写作 + 视频制作”的一体化实战项目：先生成完整故事，再拆成视频脚本和分镜，随后补齐结构化数据、字幕和角色设定，最终合成可预览的视频成片。这个案例说明 Codex 不只是能回答问题，而是能把内容创作拆成可执行、可验证、可复用的生产流程。

产物清单

这一集生成了哪些文件

  • 01-小说正文.md：完整故事文本
  • 02-视频脚本.md：短视频叙事脚本
  • 03-html-video分镜文案.md：逐镜头文案
  • 04-视频结构化数据.json：场景结构化数据
  • 05-字幕.srt：基础字幕
  • 06-角色一致性设定.md：人物外观设定
  • hf-episode-01/index.html：成片工程
  • episode-01-review.mp4：review 成片
