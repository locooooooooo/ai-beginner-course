为什么要学

核心问题

为什么同样是“AI 模型”，有人用来写代码，有人用来做图，有人用来做视频？

先记住

本课只抓 3 件事

  • OpenAI GPT 家族偏通用、多模态、工具调用
  • DeepSeek 当前公开主推 V4-Flash / V4-Pro，偏推理、代码、Agent 流程
  • Claude 偏长文本理解与写作
  • Gemini 偏谷歌生态整合与多模态入口
  • 图像模型和视频模型属于大模型，但不是传统 LLM
想明白

第一性拆解

拆解 1先按输入输出对象拆：文字、图片、视频、语音不是同一类问题。
拆解 2再按业务目标拆：有的模型偏通用协同，有的偏推理代码，有的偏视觉生成。
拆解 3最后按接入方式、成本与效果平衡来选，而不是按热度选。
可迁移

最小原理

原理 1本质一：不同模型首先是处理对象不同。
原理 2本质二：模型强项来自训练目标和产品定位，不来自宣传词。
原理 3本质三：没有“万能最优模型”，只有“当前任务更合适的模型”。
怎么讲

讲解结构

  • 主流模型家族的定位

看案例

真实案例

  • GPT 做通用问答、代码和 Agent 编排
  • DeepSeek 做推理问答和代码任务
  • gpt-image-1 做商品图和广告图
  • Seedance 2.0 做短视频和镜头生成
马上用

应用方法

方法 1先判断任务是文本、图像、视频还是多模态混合任务。
方法 2再看任务最需要的是推理、创作、速度还是系统协同。
方法 3最后再比较是否能稳定接入你的工具链和业务流程。
避坑

反常识提醒

误区 1只按排行榜选模型。
误区 2把图像模型当成 LLM 使用。
误区 3不看任务类型，直接问“哪个模型最好”。
