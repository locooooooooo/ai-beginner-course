深度讲稿

人话解释

Skill 不是一段高级提示词，而是把一类任务的做法沉淀成可复用资产。它应该包含适用场景、操作步骤、模板、参考资料、脚本和验收清单，让下次同类任务更稳定。

先区别提示词和 Skill

提示词解决这一次怎么问，Skill 解决以后同类任务怎么稳定做。

Skill 要带验收标准

只有流程没有检查点，复用时还是会跑偏；验收清单是 Skill 的核心部分。

从真实任务中迭代

不要凭空写 Skill，先做几次真实任务，把成功步骤和常见错误沉淀进去。

跟做案例

设计商品详情页文案 Skill

  1. 定义适用场景：新品详情页初稿生成。
  2. 准备输入模板：商品参数、人群、卖点、禁用词、品牌语气。
  3. 写执行步骤：资料检查、文案生成、事实校验、人工改稿。
  4. 写验收清单：规格、价格、承诺、格式、渠道要求。
完整示范

从差输入改到可用输入

差输入

写个商品文案 Skill。

好输入

请设计一个“商品详情页文案生成”最小 Skill。包含：适用场景、不适用场景、输入模板、执行步骤、输出格式、事实检查清单、常见错误、迭代方式。面向电商运营新手。

输出样例

Skill 包含输入表：商品名、规格、卖点、受众、禁用词；输出为标题、卖点、详情正文、注意事项；检查规格和承诺是否与资料一致。

检查点
  • 是否能被别人直接复用。
  • 是否写清不适用场景。
  • 是否有明确检查和更新方法。
检查理解

学完后先回答这 3 个问题

1这个 Skill 适用于哪类任务？
2它是否包含模板和验收清单？
3它如何根据真实失败案例更新？
先怎么学

按这节课自己的任务跑 4 步

每一课的学习路线都不同。先抓本课最关键的问题，再用案例和产物证明自己真的会用。

1. Skill 与提示词区别 先把本课对象画成链路：Skill 与提示词区别。标出输入、处理、输出和反馈。
2. 一个 Skill 的组成 再把关键点拆成系统部件：Skill 固化 SOP、模板、脚本、参考资料和验收标准。不要把所有责任都塞给模型。
3. 真实 Skill 案例 套进“openai-docs”这个任务，明确哪一步由 AI 做，哪一步由工具或人完成。
4. 电商如何设计 Skill 最后补验收、权限和失败处理：优先挑高频、规则明确、出错成本高的任务做 Skill。 本课产物：为“商品详情页文案生成”设计一个最小可用 Skill 结构。
先记住

本课只抓 3 件事

  • Skill 固化 SOP、模板、脚本、参考资料和验收标准
  • Skill 解决的是“下次同类任务怎么稳定复用”

想明白

第一性拆解

拆解 1把知识和流程拆开看：知道怎么做，不代表每次都能稳定做出来。
拆解 2Skill 的存在是为了把经验从人脑和一次性聊天中抽离出来。
拆解 3所以 Skill 的本质是标准化复用，而不是让提示词更花哨。
可迁移

最小原理

原理 1本质一：重复任务必须沉淀方法资产。
原理 2本质二：稳定流程比灵感更可扩展。
原理 3本质三：可验证标准是 Skill 的一部分，不是附属品。

避坑
