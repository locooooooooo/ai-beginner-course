先记住

本课只抓 3 件事

  • Git 负责记录本地修改和提交历史
  • GitHub 负责远端托管、协作和 GitHub Pages 发布
  • Codex 可以辅助检查 diff、提交、推送和验证线上链接
想明白

第一性拆解

拆解 1先把文件流向拆清楚：源码、生成结果、本地仓库、远端仓库、线上站点。
拆解 2Git 解决记录和回滚，GitHub 解决远端托管和发布，Codex 解决执行和检查。
拆解 3真正完成不是“我改了”，而是线上链接能访问到新结果。
可迁移

最小原理

原理 1本质一：Git 是可回退的修改记录。
原理 2本质二：GitHub Pages 发布的是远端仓库里的内容。
原理 3本质三：线上验证是发布闭环的最后一步。
