第14课：什么是Agent

理解 Agent 为什么比普通聊天模型更接近“会做事的人”。

返回课程总览 查看 PPT 结构总览

核心问题

为什么只靠聊天模型，很多任务永远停留在“给建议”，而 Agent 能走向“交结果”？

本课重点

  • Agent 至少包含目标、模型、工具、记忆、权限、反馈
  • Agent 能规划、执行、观察并修正
  • 真正难的是治理而不是演示

第一性拆解

拆解 1先把“会回答”和“会执行”区分开。
拆解 2Agent 的第一性价值在于把模型能力嵌进执行闭环。
拆解 3所以 Agent 的关键不是模型多聪明，而是它能否观察结果并修正动作。

最小原理

原理 1本质一：Agent 是系统，不是单一模型。
原理 2本质二：执行闭环比单次回答更重要。
原理 3本质三：Agent 越能动手，越需要治理。

讲解结构

  • Agent 和聊天模型的区别
  • Agent 的六个部件
  • 单 Agent 与多 Agent
  • 治理与边界

真实案例

  • 编程 Agent
  • 客服 Agent
  • 运营 Agent

应用方法

方法 1设计 Agent 时先列六件事：目标、模型、工具、记忆、权限、反馈。
方法 2优先做能清楚验证的单 Agent 场景。
方法 3多 Agent 只有在分工明确时才有价值。

反常识提醒

误区 1把 Agent 当成会自动成功的黑盒。
误区 2先堆多个 Agent，再考虑协作逻辑。
误区 3只看执行能力，不看权限边界。
