深度讲稿

把这一课讲透：先翻译，再演示，再提问

人话解释

LLM 负责理解和生成，Agent 负责围绕目标执行，MCP 负责连接外部工具和数据，Skill 负责把可复用做法沉淀下来。四个词合在一起，才接近一个能落地的 AI 工作系统。

用分工而不是定义解释名词

不要先让学生背 LLM、Agent、MCP、Skill 的标准定义。先问：一个 AI 要真正帮我发文章，需要理解内容、调用发布工具、读取素材库、复用发布 SOP。四个概念就是在这条链路里分工。

把四层画成一条业务链路

LLM 生成草稿，Agent 决定下一步，MCP 连接 CMS 或商品库，Skill 固化标题检查、敏感词检查、发布前验收。学生看到链路后，名词就不再孤立。

强调缺一层会发生什么

只有 LLM，容易停在建议；没有 MCP，碰不到真实系统；没有 Skill，下次同类任务又要从零解释；没有 Agent，就缺少连续执行和反馈。

跟做案例

自动生成并发布一篇新品文章

  1. LLM：根据商品资料生成文章初稿。
  2. Agent：按发布目标拆任务，决定先查资料、再写稿、再检查。
  3. MCP：连接商品库、CMS、图片素材库。
  4. Skill：复用团队的标题规则、品牌语气和发布验收清单。
完整示范

从差输入改到可用输入

差输入

LLM、Agent、MCP、Skill 都是 AI 工具，差不多都是让 AI 更强。

好输入

LLM 负责理解和生成，Agent 负责围绕目标连续执行，MCP 负责连接外部系统和数据，Skill 负责把一类任务的做法沉淀成可复用资产。它们是同一条工作链路里的不同分工。

输出样例

业务例子：自动发布新品文章。LLM 写初稿，Agent 拆解查资料、写稿、检查、发布草稿的步骤，MCP 连接商品库和 CMS，Skill 固化标题规则、品牌语气和发布前检查清单。

检查点
  • 是否用分工解释名词，而不是堆定义。
  • 是否能把四个概念放进同一个真实流程。
  • 是否说明缺少某一层会出现什么问题。

先记住

本课只抓 3 件事

  • 大模型是 AI 中的一类基础模型
  • LLM 是擅长语言理解与生成的大模型
  • Agent 是会调工具、会执行流程的系统
  • MCP 解决连接外部系统的问题
  • Skill 解决工作方法沉淀的问题

可迁移

最小原理

原理 1本质一：名词存在的意义是分工，不是炫技。
原理 2本质二：业务落地一定同时涉及理解、执行、连接、复用四件事。
原理 3本质三：单个模型能力不等于完整系统能力。
