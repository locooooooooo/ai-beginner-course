为什么要学

核心问题

为什么很多人学了一堆名词，最后还是不会落地？因为不知道这些概念各自负责哪一段。

先怎么学

先用 4 步把这课跑起来

这不是标题清单，是给新手的跟做路线。你不需要先记住所有名词，先把任务做完再回头理解术语。

1. 先说人话 用自己的话回答：这节课到底解决什么困惑？可以先照着核心问题复述一遍：为什么很多人学了一堆名词，最后还是不会落地？因为不知道这些概念各自负责哪一段。
2. 只抓三句 先记住前 3 个重点，不急着背完整定义：大模型是 AI 中的一类基础模型；LLM 是擅长语言理解与生成的大模型；Agent 是会调工具、会执行流程的系统。
3. 套进场景 从案例里挑一个最熟的场景，例如“让大模型写文案”，写出输入、AI 做什么、人工怎么检查。
4. 留下产物 完成本课产物：用一句话分别解释 LLM、Agent、MCP、Skill 的作用，再举一个你能想到的真实业务例子。
先记住

本课只抓 3 件事

  • 大模型是 AI 中的一类基础模型
  • LLM 是擅长语言理解与生成的大模型
  • Agent 是会调工具、会执行流程的系统
  • MCP 解决连接外部系统的问题
  • Skill 解决工作方法沉淀的问题
想明白

第一性拆解

拆解 1先问每个概念解决的最小问题是什么，而不是先背定义。
拆解 2LLM 解决理解与生成问题，Agent 解决执行问题，MCP 解决连接问题，Skill 解决复用问题。
拆解 3把四者放回同一条业务链路，才知道为什么缺一块就落不下去。
可迁移

最小原理

原理 1本质一：名词存在的意义是分工，不是炫技。
原理 2本质二：业务落地一定同时涉及理解、执行、连接、复用四件事。
原理 3本质三：单个模型能力不等于完整系统能力。

马上用

应用方法

方法 1碰到新概念时，先问它解决的是理解、执行、连接还是复用。
方法 2做系统设计时，按四层分工思考，而不是一股脑扔给模型。
方法 3画系统图时，把这四层拆开写清楚。
避坑

反常识提醒

误区 1把 Agent 和模型混成一个概念。
误区 2以为接了 MCP 就自动有业务流程。
误区 3把 Skill 当成高级提示词，而不是可复用资产。
