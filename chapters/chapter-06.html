本章学习路线

按顺序学更稳；已经会的课也建议至少填完工作纸。

第 21 课

Codex能做什么

理解 Codex 不是普通聊天机器人，而是能进入项目现场读、改、跑、验收的 AI 工作助手。

本课产物 写一个给 Codex 的任务说明，包含目标、范围、验收标准和禁止事项。
主讲义工作纸
第 22 课

Loop Engineering是什么

理解 Loop Engineering 是一种循环式工作方法：计划、执行、验收、修正，再进入下一轮。

本课产物 为“把一个旧网页改成手机友好教程站”设计 3 轮 Loop，每轮写清目标和验收证据。
主讲义工作纸
第 23 课

如何用Codex制作好看的网页

学会把目标用户、页面结构、视觉规范和移动端验收交给 Codex，让网页既能看又能用。

本课产物 写一段网页制作需求，让 Codex 为“AI 新手课程”设计一个手机优先首页，并列出 5 条验收标准。
主讲义工作纸
第 24 课

Git、GitHub与Codex协作

理解 Git 记录本地修改，GitHub 托管和发布代码，Codex 可以帮助提交、推送和验证线上结果。

本课产物 写出一次 GitHub Pages 发布流程：本地生成、检查、提交、推送、线上验证，每一步写出成功证据。
主讲义工作纸
第 25 课

你真的会用Dify吗？

学会用 Codex 搭建 Dify 平台、完成 Docker 部署，并把 Wiki 发布、Bug 汇总和自动 Code Review 做成自动化流程。

本课产物 设计一个 Dify + Codex 自动化方案：先写 Docker 部署步骤，再画出 Wiki 发布、Bug 飞书文档、自动 Code Review 三条流程。
主讲义工作纸
